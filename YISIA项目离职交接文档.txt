一、项目概述
================================================================================

YISIA 是一个面向医美行业的 AI 智能对话系统，定位为用户的"美丽闺蜜"。
系统集成了 RAG（检索增强生成）、知识图谱、内容审核、多语言国际化、
用户画像、自进化等高级功能，采用微服务架构，由三个独立服务组成。

1.1 系统架构总览
--------------------------------------------------------------------------------

系统由三个微服务组成，部署在同一台 Linux 服务器上，通过 HTTP 内部通信：

  服务一：algorithm_services（核心算法服务）
    技术栈：Python / FastAPI / Uvicorn
    端口：6732
    systemd 服务名：yisia-algorithm
    职责：核心算法服务，负责对话理解、功能规划、内容审核、LLM 调用等全部 AI 能力

  服务二：knowledge_base_service（知识库服务）
    技术栈：Python / FastAPI / Uvicorn
    端口：8002
    部署方式：手动管理（无 systemd 服务）
    职责：知识库 RAG 检索、向量索引（HNSW）、知识图谱、重排序

  服务三：frontend_services（前端静态服务）
    技术栈：Node.js / Express
    端口：3000
    systemd 服务名：yisia-frontend
    职责：SPA 单页应用、用户认证、流式对话界面

1.2 请求处理流程
--------------------------------------------------------------------------------

用户输入
  → 步骤1：快速短路检测（词表匹配简单问候/确认/感谢）
  → 步骤2：路由决策（判断 need_plan / need_search，同时异步并行启动内容违规检测）
  → 若 need_plan=False 则直接走 free_chat 兜底
  → 若 need_plan=True 则进入步骤3：功能规划（决定调用哪些功能模块）
  → 步骤4：执行功能（按顺序执行规划的功能，结果存入 session）
  → 步骤5：自动选择 Chat 方式（有知识检索结果用 knowledge_chat，无结果用 free_chat）
  → 步骤6：SSE 流式输出返回结果

三、技术栈详情
================================================================================

3.1 后端技术栈
--------------------------------------------------------------------------------

  语言：Python 3.11 / 3.13
  Web 框架：FastAPI（异步 Web 框架）
  ASGI 服务器：Uvicorn
  HTTP 客户端：httpx（异步）+ requests（同步）
  数据验证：Pydantic
  环境变量：python-dotenv
  LLM 服务商：阿里云 DashScope（通义千问系列）
  Embedding：text-embedding-v3（1024 维向量）
  向量索引：HNSW（自实现，M=16, ef_construction=200, ef_search=50）
  部署：Linux systemd 守护进程

3.2 前端技术栈
--------------------------------------------------------------------------------

  语言：JavaScript（ES6+），原生 JS，无框架
  Web 服务器：Node.js + Express 4.18.2
  开发工具：nodemon 3.0.1
  Markdown 渲染：marked.js 12.0.1（CDN 引入）
  代码高亮：highlight.js 11.9.0（CDN 引入）
  架构：SPA（单页应用），对象字面量模式
  通信协议：SSE（Server-Sent Events）流式聊天
  UI 特性：6 种主题、20+ 种语言、响应式布局、暗色模式

3.3 AI/算法技术栈
--------------------------------------------------------------------------------

  RAG：自研 RAG 系统（向量检索 + BM25 + 实体检索 + 知识图谱）
  知识图谱：自研（实体关系抽取 + 图谱构建 + 图谱增强 RAG）
  文档分块：HybridChunker（语义相似度分块 + 父子分块策略）
  内容审核：多层审核（题库匹配 C35-C39 + LLM 检测 C41 + 历史违规库 C42）
  国际化：BCP 47 标准，支持 20+ 语言自动检测
  用户画像：用户风格学习、知识图谱、记忆挖掘

4.1 algorithm_services API（端口 6732，root_path=/chat）
--------------------------------------------------------------------------------

  统一入口：POST /api/v1/entrance
    说明：聊天机器人主接口（SSE 流式）
    请求体：
    {
      "session_id": "996",
      "user_id": "996",
      "lang": "zh-CN",
      "stream_flag": true,
      "user_input": "用户输入内容",
      "context": "[{\"user\":\"输入1\",\"response\":\"回答1\"}]",
      "minor_mode": false,
      "personalize": true
    }
    SSE 响应码：
      102 - 中间过程（AI 思考中）
      200 - 最终结果
      300 - 流式打字（当前忽略）
      403 - 违规内容拦截
      500 - 错误

  功能路由（14个）：
    POST /api/v1/dialog-summary/*          # 对话摘要
    POST /api/v1/entity-relation-extraction/*  # 实体关系联合抽取
    POST /api/v1/free-chat/*               # 闲聊
    POST /api/v1/function-planner/*        # 功能规划器
    POST /api/v1/intent-clarify/*          # 意图澄清
    POST /api/v1/intent-recognize/*        # 意图识别
    POST /api/v1/text-summary/*            # 文本摘要
    POST /api/v1/title-generation/*        # 标题生成
    POST /api/v1/content-moderation/*      # 内容审核
    POST /api/v1/llm-web-search/*          # LLM 联网搜索
    POST /api/v1/user-style/*              # 用户风格学习
    POST /api/v1/user-knowledge-graph/*    # 用户知识图谱

  监控指标：
    GET  /api/v1/metrics/hourly            # 小时级统计

9.1 登录视图（loginView.js）

十、关键设计亮点
================================================================================

10.1 多 LLM 服务商容错
--------------------------------------------------------------------------------

  Key 轮转：每个服务商支持最多 9 个 API Key，400/403 错误时自动切换
  模型回退：配额不足时自动切换到备用模型
  供应商回退：主供应商失败时切换到备用供应商
  启动时 Key 健康检测：并行检测所有 Key 的可用性和支持的模型

10.2 内容审核多层防护
--------------------------------------------------------------------------------

  第一层：关键词检测（7 类违规内容，多线程并行）
  第二层：LLM 检测（C41，qwen-flash 模型，阈值 0.85）
  第三层：题库匹配（C35-C39，Embedding 相似度，阈值 0.85）
  第四层：历史违规库匹配（C42，Embedding 相似度，阈值 0.90）

10.3 SSE 流式输出
--------------------------------------------------------------------------------

  协议：Server-Sent Events（text/event-stream）
  状态码：102（处理中）、300（流式片段）、500（错误）、200（完成）
  前端：chatApi.js 使用 Fetch API + ReadableStream 接收

10.4 会话上下文管理
--------------------------------------------------------------------------------

  CONTEXT_COLLAPSE 归档机制：当对话轮数超过 MAX_LOOP（默认 2）时，
  将早期对话归档为摘要，保留最近轮次的完整上下文

10.5 性能监控和指标统计
--------------------------------------------------------------------------------

  性能监控：@monitor_async 装饰器跟踪异步函数执行时间
  指标管理：metrics_manager 记录 LLM 调用成本、请求级别统计
  监控面板：/admin 页面提供实时监控仪表盘
  指标 API：小时级、日级、实时统计 API
